Funksjon som sender kall til WFS-tjenesten

  • url: url til WFS-tjeneste
  • request: XML tekst-streng, kodet som byte-string med encode-funksjon
  • outfile: filnavn p XML/GML-fil
In[2]:
def wfs_request(url, request, outfile):
    response = requests.post(url, request)
    doc = xml.dom.minidom.parseString(response.content.decode())
    xml_response = doc.toprettyxml()
    with open(outfile, 'w', encoding = 'utf-8') as f:
        f.write(xml_response)

GetCapabilities-kall

Utformingen av dette kallet er helt uavhengig av innholdet i tjenesten. Det brukes derfor samme kall til alle tjenester. Vi er spesielt interessert i resultatet under FeatureTypeList.

In[4]:
getcapabilities = """<?xml version="1.0" encoding="UTF-8"?>
<GetCapabilities
  service = "WFS"
  version = "2.0.0"
  xmlns = "http://www.opengis.net/wfs/2.0"
  xmlns:xsi = "http://www.w3.org/2001/XMLSchema-instance"
  xsi:schemaLocation = "http://www.opengis.net/wfs http://schemas.opengis.net/wfs/2.0/wfs.xsd"/>
""".encode()

DescribeFeatureType

Dette kallet tilpasses den aktuelle tjenesten basert p resultatet av GetCapabilities-kallet.

  • XML namespace, her: xmlns:app="http://skjema.geonorge.no/SOSI/produktspesifikasjon/Vannkraft/1.3"
  • TypeName, her: app:InnsjRegulert

Resultatet av kallet beskriver hvilke egenskaper (attributter) den aktuelle objekttypen (TypeName) har.

In[5]:
describefeaturetype = """<?xml version="1.0" encoding="UTF-8"?>
<DescribeFeatureType
  service = "WFS"
  version = "2.0.0"
  xmlns = "http://www.opengis.net/wfs/2.0"
  xmlns:app ="http://skjema.geonorge.no/SOSI/produktspesifikasjon/Vannkraft/1.3"
  xmlns:xsi = "http://www.w3.org/2001/XMLSchema-instance"
  xsi:schemaLocation = "http://www.opengis.net/wfs/2.0 http://schemas.opengis.net/wfs/2.0/wfs.xsd">
  <TypeName>app:InnsjRegulert</TypeName>
</DescribeFeatureType>
""".encode()

GetFeature - uten Filter Encoding

Dette kallet henter objekter fra tjenesten. Namespace og Typename m spesifiseres. Her er det ikke noen spesifisering av hvordan objekter skal velges ut. Resultatet blir derfor tilfeldig, men det kan vre en frste test p f ut et objekt fra tjenesten. Kallet spr i utgangspunktet etter alle objekter av den aktuelle objekttypen. Det er derfor viktig begrense antallet objekter med count. Legg merke til:

  • wfs:Query
  • typeNames
  • srsName
In[6]:
getfeature = """<?xml version="1.0" encoding="UTF-8"?>
<GetFeature
  service = "WFS"
  version = "2.0.0"
  count = "1"
  xmlns = "http://www.opengis.net/wfs/2.0"
  xmlns:app="http://skjema.geonorge.no/SOSI/produktspesifikasjon/Vannkraft/1.3">
  <Query typeNames = "app:InnsjRegulert" srsName="urn:ogc:def:crs:EPSG::25832">
  </Query>
</GetFeature>
""".encode()

GetFeature - med Filter Encoding

Dette kallet er utvidet med et fes:Filter-element. Filteret er av type BBOX som er oppgitt som en gml:Envelope. fes:ValueReference henviser til omrde som er navnet p geometri-egenskapen i objekttypen. Denne opplysningen kunne vi finne basert p resultatet av DescribeFeatureType-kallet, under InnsjRegulert. Legg merke til at kallet ogs er utvidet med to nye namespaces:

  • xmlns:fes, for Filter Encoding
  • xmlns:gml, for GML
In[7]:
getfeatureBBOX = """<?xml version="1.0" encoding="UTF-8"?>
<GetFeature
  version = "2.0.0"
  xmlns = "http://www.opengis.net/wfs/2.0"
  xmlns:fes = "http://www.opengis.net/fes/2.0"
  xmlns:gml="http://www.opengis.net/gml/3.2"
  xmlns:app="http://skjema.geonorge.no/SOSI/produktspesifikasjon/Vannkraft/1.3">
  <Query typeNames = "app:InnsjRegulert"  srsName="urn:ogc:def:crs:EPSG::25832">
    <fes:Filter>
      <fes:BBOX>
        <fes:ValueReference>omrde</fes:ValueReference>
        <gml:Envelope srsName="urn:ogc:def:crs:EPSG::25832">
          <gml:lowerCorner>586500 6735500</gml:lowerCorner>
          <gml:upperCorner>596500 6745500</gml:upperCorner>
        </gml:Envelope>
      </fes:BBOX>
    </fes:Filter>
  </Query>
</GetFeature>
""".encode()
